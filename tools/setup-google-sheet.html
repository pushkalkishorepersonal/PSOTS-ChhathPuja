Google Sheet Setup Guide

Follow these steps to connect your PSOTS website to a Google Sheet for live data management

1 Create a New Google Sheet

  1. Go to sheets.google.com
  2. Click + Blank spreadsheet
  3. Name it: PSOTS Chhath Puja Data
Open Google Sheets →

2 Open Apps Script Editor

  1. In your new Google Sheet, click Extensions in the menu bar
  2. Click Apps Script
  3. A new tab will open with the script editor
  4. Delete any default code in the editor (select all → delete)

3 Paste the Backend Code

4 Run Initial Setup

  1. In Apps Script editor, find the function dropdown at the top (says "myFunction" or similar)
  2. Change it to setupSheets
  3. Click the ▶ Run button
  4. When prompted, click Review Permissions → choose your Google account → click Advanced → Go to PSOTS Chhath Puja Data (unsafe) → Allow
  5. You'll see a success message: "✅ Setup Complete!"

5 Import Historical Data

  1. In the script editor, click + next to "Files" on the left sidebar
  2. Click Script and name the new file ImportHistory
  3. Copy and paste the contents from the button below into this new file
  4. Select importAllHistory from the function dropdown
  5. Click ▶ Run
  6. Click Yes if prompted about existing data
  7. Wait for "✅ Import Complete!" message (imports 429 records)
Click the button above to load the import script.

6 Deploy as Web App

  1. In Apps Script editor, click Deploy (blue button, top right)
  2. Click New deployment
  3. Click the ⚙️ gear icon next to "Select type" → choose Web app
  4. Fill in:
    • Description: PSOTS Backend v1
    • Execute as: Me (your email)
    • Who has access: Anyone
  5. Click Deploy
  6. Copy the Web app URL — it looks like:
    https://script.google.com/macros/s/ABCDEF.../exec

7 Connect to Your Website

  1. Open your PSOTS website → go to Admin Panel
  2. Click the Settings tab
  3. Paste the Web App URL in the "Apps Script URL" field
  4. Click Save Settings
  5. Refresh the page — your live data should now load!

⚠️ Important Notes

  • The Google Sheet must remain accessible (don't delete it!)
  • If you update the Apps Script code, you must create a new deployment for changes to take effect
  • The script runs under your Google account — only you can see the raw data in the Sheet
  • Everyone accessing the website sees the same data (it's global, not per-device)
  • For future contributions: residents submit via the website → data goes to your Google Sheet → admin verifies in the Sheet or Admin panel
